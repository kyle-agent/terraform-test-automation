# Contributing

이 리포는 회귀 회피 시스템이 본업입니다. 변경 사항은 1) 새 회귀 테스트 추가, 2) helper 보강, 3) CI 워크플로우 개선의 세 카테고리만 다룹니다.

## 변경 단위

- 한 PR = 하나의 sub-issue에 대응하는 한 개의 회귀 테스트 (+ 필요한 helper/시나리오)
- helper 변경은 단독 PR로 분리 (assertion 시그니처 변경은 기존 테스트에도 영향)

## 코드 스타일

- Go 1.22, `go fmt ./...` 적용
- 테스트는 가능한 한 표준 라이브러리 + `tests/common`만 사용 (제3자 의존성 최소화)
- 가독성 > 짧음. 단언 한 줄이 무슨 결함을 잡는지 주석 한 줄로 설명
